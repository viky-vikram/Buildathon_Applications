# Daybook Leave Management

Daybook is a buildathon leave-management app with the architecture requested in the build plan:

- FastAPI backend for auth, role permissions, leave workflow, employees, analytics, calendar, and records.
- Streamlit frontend that keeps the prototype Daybook styling while calling the API for all reads and writes.
- SQLite primary store at `data/daybook.sqlite3`.
- Excel record workbook at `database excel/daybook_records.xlsx`, with append/rebuild support for Employees, LeaveRequests, Decisions, Balances, PolicyDates, and ActivityEvents.
- Shared demo fixtures and UI theme live in `shared/` so backend and frontend do not duplicate constants.

## Project Layout

```text
backend/
  app/
    main.py                 # FastAPI app, CORS, health, router registration
    database.py             # SQLite engine/session setup
    models.py               # SQLAlchemy ORM models
    schemas.py              # Pydantic API schemas
    deps.py                 # Auth and role dependencies
    routers/                # Auth, employees, leave, approvals, balances, calendar, analytics, records
    services/               # Leave policy, workflow, employee creation, Excel writer
    seed.py                 # Demo seed/reset command
frontend/
  api_client.py             # Central Streamlit API helper
  streamlit_app.py          # API-backed Streamlit app
shared/
  demo_data.py              # Demo users, seed fixtures, labels, leave types
  theme.py                  # Streamlit theme injection
data/
  daybook.sqlite3           # Generated by seed command
database excel/
  daybook_records.xlsx      # Employee details and API record workbook
tests/                      # Backend API regression tests
```

## Run Locally

From the project root:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m backend.app.seed
uvicorn backend.app.main:app --reload --port 8000
```

Open a second terminal in the same activated environment:

```powershell
streamlit run frontend/streamlit_app.py
```

Then open Streamlit at `http://localhost:8501`. The frontend expects the API at `http://127.0.0.1:8000`; override with:

```powershell
$env:DAYBOOK_API_BASE_URL="http://127.0.0.1:8000"
```

For a deployed frontend, set allowed API origins with a comma-separated value:

```powershell
$env:DAYBOOK_CORS_ORIGINS="https://your-streamlit-app.example"
```

## API

FastAPI docs are available after starting the API:

```text
http://127.0.0.1:8000/docs
```

Core endpoints include:

- `POST /auth/login`, `GET /auth/me`, `POST /auth/logout`
- `GET /employees`, `POST /employees`, `GET /employees/{id}`, `PATCH /employees/{id}`, `GET /employees/{id}/balances`
- `GET /leave-requests`, `POST /leave-requests`, `GET /leave-requests/{id}`, `POST /leave-requests/{id}/cancel`
- `GET /approvals/pending`, `POST /leave-requests/{id}/approve`, `POST /leave-requests/{id}/reject`
- `GET /calendar/wallchart`, `GET /calendar/policy-dates`, `POST /calendar/policy-dates`
- `GET /analytics/dashboard`, `GET /analytics/team`, `GET /analytics/org`, `GET /analytics/burnout`
- `GET /records/excel/status`, `POST /records/excel/rebuild`

## Storage

Seed/reset creates:

```text
data/daybook.sqlite3
database excel/daybook_records.xlsx
```

SQLite is the source of truth. New employee records, leave requests, approvals, rejections, cancellations, policy dates, balance rows, and activity events are appended to Excel. HR can view and rebuild the workbook from the Streamlit Records page or via `/records/excel/rebuild`.
